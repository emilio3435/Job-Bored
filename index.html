Free, ~3 minutes. JobBored is BYO keys — your Client ID stays in your browser. We’ve copied this site’s origin to your clipboard so you can paste it when Google asks.

Step-by-step guide (open if it’s your first time)
  1. In Google Cloud Console, create or pick a project (top bar → project picker → New project).
  2. Configure the OAuth consent screen (left nav → APIs & Services → OAuth consent screen). Pick External, fill in the app name + your email, save. You can add yourself as a test user under Audience to skip “unverified app” warnings.
  3. Enable the Google Sheets API and Google Drive API for the project (APIs & Services → Library → search → Enable).
  4. Open Credentials → Create credentials → OAuth client ID → application type Web application.
  5. Under Authorized JavaScript origins, click Add URI and paste the origin shown above (already on your clipboard). Leave redirect URIs empty — JobBored doesn’t use them.
  6. Click Create. Google shows a popup with your Client ID — copy it and paste below.

Stuck? The Client ID always ends in .apps.googleusercontent.com. If you see a “redirect_uri_mismatch” later, you picked the wrong app type — recreate as Web application.

    Set up the job posting scraper

    Cheerio runs on your machine (Node.js) so this page can read job descriptions that aren’t available from the browser alone. From the repo root, one command starts both the dashboard and the scraper.

    Plug and play: in the project folder (where package.json lives), run npm install once, then npm start every time. That installs server dependencies automatically and starts the UI on http://localhost:8080 plus the scraper on 127.0.0.1:3847.
    GitHub Pages (HTTPS): the dashboard is served over HTTPS, so the browser will not call http://127.0.0.1 on your laptop (mixed content). For Fetch posting from Pages, deploy the server/ app to Render, Fly, Railway, etc., paste the HTTPS base URL in Settings, or run the UI locally with npm start on http://localhost. See DEPLOY-SCRAPER.md.
    1. Install Node.js (if needed)

      Download the LTS build from nodejs.org. Then confirm in a terminal: 

      node -v
    2. Start dashboard + scraper (project root)

      Terminal / PowerShell in the project folder — first time installs everything (including the server package): 

      npm install
npm start

      Leave this running. Logs should show the web server on port 8080 and [job-scraper] http://127.0.0.1:3847.

      Scraper only (no dashboard)

      cd server && npm install && npm start — same API on port 3847.

    3. Confirm the URL in Settings

      You can leave the scraper URL blank on localhost — the app uses http://127.0.0.1:3847 automatically. Otherwise paste your URL below, then click Save & reload in Settings. 

      http://127.0.0.1:3847
    4. Use it on a job card

      Open a role with a Link, expand the card, and click Fetch posting. With a Gemini key, you also get AI fit angle and talking points from the scraped text.

    Connection refused / Failed to fetch? The scraper isn’t running. In a terminal: cd server then npm start and leave that window open. HTTPS dashboard? Browsers may block http://127.0.0.1 from a secure site — use plain HTTP locally or deploy the scraper with HTTPS. Blocked job page? Some sites block bots; try an employer career page. See SETUP.md.
