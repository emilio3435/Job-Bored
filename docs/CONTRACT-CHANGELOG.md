# Contract changelog

Human-readable history of changes to **[AGENT_CONTRACT.md](../AGENT_CONTRACT.md)**, **[schemas/](../schemas/)**, and **[examples/](../examples/)**. For the roadmap (fixtures, CI, pipeline schema), see **[CONTRACT-HARDENING-PLAN.md](CONTRACT-HARDENING-PLAN.md)**.

| Date (UTC) | Change                                                                                                                                                                                             | Compatibility                               |
| ---------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------- |
| 2026-06-18 | Added the **pipeline-update** contract: `schemas/pipeline-update-request.v1.schema.json`, fixture `examples/pipeline-update-request.v1.json`, worker endpoint `POST /pipeline-update`, and `npm run test:pipeline-update-contract` (wired into `test:contract:all`). Lets an external agent advance an existing Pipeline row (Status, Contact, Notes, Last contact, Did they reply?) from inbound signals; matched by job URL with company+title fallback; local-first auth via `x-discovery-secret`; writes use the worker's own Google credential. | Additive — new interface contract; existing receivers unaffected. |
| 2026-06-04 | Documented the optional **`mergedUserProfile`** top-level field on the discovery webhook request. It was already sent by the dashboard (`discovery-readiness.js`) and typed in `contracts.ts`, but was missing from the schema and `AGENT_CONTRACT.md`. Non-secret (no raw resume text); validated via ajv and never persisted. | Additive — field was already optional; documentation/schema now match the wire. |
| 2026-05-26 | Added optional `trigger`, `discoveryProfile.profileSnapshot`, and `discoveryProfile.searchPlan` to discovery webhook v1. The trigger enum covers manual, browser-tab, local, GitHub, Cloudflare, Apps Script, and CLI origins. The snapshot records non-secret current profile/resume/preferences/schedule metadata, and the search plan carries the deterministic rotated query bundle used by manual and scheduled runs. | Additive — fields are optional and old receivers can ignore them. |
| 2026-05-20 | Documented hosted discovery run status polling: hosted `/runs/:runId` endpoints require the returned `statusPath` with its per-run `statusToken`, or an equivalent `x-run-status-token`/webhook secret. Browser clients and relays must preserve the returned `statusPath` exactly, including query parameters. | Additive — async webhook acknowledgements remain compatible; hosted status polling is explicitly authenticated. |
| 2026-05-20 | Added `Expired` as a Pipeline status enum and documented safe expired-job cleanup semantics: move confirmed closed rows to column M `Expired`, append audit details to Notes, and never delete rows. | Additive — existing statuses and columns remain valid. |
| 2026-05-11 | v2 `.kanban-card` now emits read-only `data-*` attributes (jd-snippet, notes, location, salary, job-url, source, applied-at, follow-up, tags, fit, replied, talking-points, contacts, company-tagline, employment) consumed by the dossier view-model in `dawn-data.js`. Contract documented in `AGENT_CONTRACT.md`; covered by `tests/dossier-card-attrs.test.mjs`. | Additive — legacy UI unaffected (off-flag path ignores the attrs). |
| 2026-05-07 | Added optional **`companyAllowlist`** and **`companyBlocklist`** top-level arrays to the discovery webhook request. `companyAllowlist` supports per-run company picker overrides and is capped at 500 entries; `companyBlocklist` suppresses companies and is capped at 50 entries. Empty/missing preserves previous behavior. Schema, examples, worker contract types, and webhook handler validation all updated together. | Additive — backward compatible (optional fields). |
| 2026-04-09 | ATS server now auto-loads `server/.env` (`dotenv`) and exposes ATS config status in `/health`; added persistent env template `server/ats-env.example`.                                             | Additive — runtime configuration hardening. |
| 2026-04-09 | Added ATS scorecard contracts: `schemas/ats-scorecard-request.v1.schema.json`, `schemas/ats-scorecard-response.v1.schema.json`, and fixtures under `examples/`. Added `npm run test:ats-contract`. | Additive — new ATS interface contract.      |
| 2026-04-08 | Added `examples/discovery-webhook-request.v1.json` and `examples/discovery-webhook-request.v1-with-profile.json` aligned with `triggerDiscoveryRun` in `app.js`.                                   | Additive — examples only.                   |
| 2026-04-08 | Documented **Webhook receiver checklist** in `AGENT_CONTRACT.md` (CORS, 2xx, OPTIONS). Linked from `SETUP.md`.                                                                                     | Additive — documentation.                   |
| 2026-04-08 | Introduced **`npm run test:contract`** — validates example JSON against `schemas/discovery-webhook-request.v1.schema.json` and checks discovery `payload` keys in `app.js` match the schema.       | Additive — CI/dev tooling.                  |
| 2026-04-08 | Added **`schemas/pipeline-row.v1.json`** and **`npm run test:pipeline-contract`** — Pipeline header row + enums aligned with README and `app.js` status/priority lists.                            | Additive — machine-readable Interface A.    |
| 2026-04-08 | Added **`npm run lint:skills`**, **`npm run test:contract:all`**, **[CONTRIBUTING.md](../CONTRIBUTING.md)** contract checklist; GitHub **Validate contract** runs the full suite.                  | Additive — contributor + CI tooling.        |

**Versioning reminder:** Breaking changes to the discovery POST body should bump **`schemaVersion`** in the schema, `app.js`, and this log, with a migration note for implementers.
